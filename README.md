## 📌 Описание проекта
**Codebattle** — это игровая платформа для соревновательного программирования, где разработчики пишут код на скорость. Проект представляет собой сложную систему с динамическим контентом и игровыми механиками.

**Моя роль:** Проведение комплексного тестирования платформы для обеспечения стабильности соревнований и корректности отображения игровых данных.

---

## 🛠 Технологический стек и инструменты
* **Инструменты:** Chrome DevTools, работа с элементами DOM, анализ сетевых запросов.
* **Методологии:** Функциональное и нефункциональное тестирование, регрессионное тестирование.
* **Документация:** YAML для структурирования тестовых артефактов и баг-репортов.

## 🎯 Область тестирования (Scope)
В данном проекте фокус был сделан на проверку взаимодействия пользователей в реальном времени:
1. **Игровой процесс:** Стабильность работы таймеров, корректность проверки решений.
2. **Безопасность (Security):** Базовые проверки на уязвимости и доступ к данным.
3. **Локализация:** Корректность отображения интерфейса на разных языках.
4. **Регресс:** Проверка того, что новые изменения не затронули базовый функционал соревнований.

## 🔍 Исследовательское тестирование (Security Note)

В рамках проверки безопасности платформы был проведен анализ изоляции среды выполнения (Sandboxing) пользовательского кода.

* **Сценарий:** Попытка выхода за пределы песочницы с помощью выполнения системных команд Python (вызов `os.listdir('/')` в редакторе кода).
* **Результат:** Получен доступ к чтению корневой директории (системные папки `/etc`, `/root`, `/bin` и др.).
* **Анализ:** После изучения архитектуры проекта установлено, что платформа использует Docker-контейнеры. Вывод отражает файловую систему временного, изолированного контейнера, а не хост-сервера.
* **Вердикт:** *Not a Bug (Expected Behavior)*. Данный кейс подтвердил надежность архитектурного решения по изоляции пользовательского кода и защите основного сервера.
